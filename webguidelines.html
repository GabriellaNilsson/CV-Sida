Web Guidelines

WCAG 3.3.4 (AA): Ge möjlighet att ångra, korrigera eller bekräfta vid viktiga transaktioner: inte tillämpbart utifrån koden

WCAG 2.4.1 (A): Gör det möjligt att hoppa förbi återkommande innehåll: inte tillämpbart utifrån koden

WCAG 2.2.1 (A): Gör det möjligt att justera tidsbegränsningar: inte tillämpbart utifrån koden

WCAG 1.4.2 (A: Gör det möjligt att pausa, stänga av eller sänka ljud: inte tillämpbart utifrån koden

WCAG 2.2.2 (A): Gör det möjligt att pausa eller stänga av rörelser: inte tillämpbart utifrån koden

WCAG 2.5.2 (A): Gör det möjligt att ångra klick: inte tillämpbart utifrån koden

WCAG 2.4.7 (AA): Gör det synligt vad som är i fokus: inte tillämpbart utifrån koden

WCAG 1.3.3 (A): Gör inte instruktioner beroende av sensoriska kännetecken: inte tillämpbart utifrån koden

WCAG 2.4.3 (A): Ha en meningsfull fokusordning: innehållet är logiskt och organiserat placerat

WCAG 3.2.1 (A): Ingen kontextförändring vid fokus: inte tillämpbart utifrån koden

WCAG 2.1.2 (A): Ingen tangentbordsfälla: inte tillämpbart utifrån koden

WCAG 1.4.13 (AA): Innehåll som visas vid hover eller fokus: Popups går att stänga.

WCAG 3.2.4 (AA): Konsekvent benämning: alla knappar har tydliga och logiska benämningar.

WCAG 2.3.1 (A): Orsaka inte epileptiska anfall genom blinkande: inte tillämpbart utifrån koden

WCAG 1.3.2 (A): Presentera innehållet i en meningsfull ordning: inte tillämpbart utifrån koden

WCAG 1.4.12 (AA): Se till att det går att öka avstånd mellan tecken, rader, stycken och ord: ingen text skyms eller överlappas, men lagom avstånd så det är läsbart.

WCAG 4.1.3 (A): Se till att hjälpmedel kan presentera meddelanden som inte är i fokus: inte tillämpbart utifrån koden

WCAG 1.3.4 (AA): Se till att innehållet anpassas efter skärmens riktning: tillämpat i koden

WCAG 4.1.2 (A): Se till att komponenter fungerar i hjälpmedel: inte tillämpbart utifrån koden

WCAG 1.4.4 (A): Se till att text går att förstora: inte tillämpat då viss innehåll skyms och blir svårt att ta del av

WCAG 2.5.3 (A): Se till att text på knappar och kontroller överensstämmer med maskinläsbara namn: inte tillämpbart utifrån koden, då hjälpmedel som röststyrning inte finns

WCAG 1.4.10 (AA): Skapa en flexibel layout som fungerar att vid förstoring eller på liten skärm: tillämpat

WCAG 3.3.2 (A): Skapa tydliga fältetiketter/ledtexter: det är tydligt och logikst vad som ska fyllas i i formuläret.

WCAG 2.4.6 (AA): Skriv beskrivande rubriker och ledtexter: det är tydliga och logiska rubriker.

WCAG 2.4.2 (A): Skriv beskrivande sidtitlar: finns tydliga sidtitlar

WCAG 2.4.4 (A): Skriv tydliga länkar: finns tydliga länkar med lagom kort text.

WCAG 1.2.3 (A): Syntolka eller erbjud alternativ till videoinspelningar: inte tillämpbart utifrån koden, då det inte finns videor på sidan

WCAG 1.2.5 (AA): Syntolka videoinspelningar: inte tillämpbart utifrån koden, då det inte finns videor på sidan

WCAG 1.2.2 (A): Texta ljudet i inspelade filmer: inte tillämpbart utifrån koden, då det inte finns videor på sidan

WCAG 3.2.2 (A): Utför inga oväntade förändringar vid inmatning: inte tillämpbart utifrån koden, då den enda inmatning som finns är formuläret och efftersom sidan är dynamisk så kan inget hända

WCAG 2.1.4 (A): Var försiktig med kortkommandon: inte tillämpbart utifrån koden, då det inte finns tagentbordskommandon.

WCAG 3.2.3 (AA): Var konsekvent i navigation: tillämpat, då det finns en och samma konskventa navigationsbar

WCAG 3.3.1 (A): Visa var ett fel uppstått och beskriv det tydligt: inte tillämpbart utifrån koden, då det inte finns felmeddelanden och formuläret inte är interaktivt.

WCAG 2.1.1 (A): All funktionalitet ska kunna användas med tangentbord: inte tillämpbart på koden.

WCAG 3.1.1 (A): Ange sidans språk i koden: tillämpat.
